# user_account
Таблица `user_account` содержит базовую информацию о пользователе, необходимую для аутентификации и управления аккаунтом. Она служит основой для создания профиля пользователя и связывания его с различными аспектами социальной сети, такими как посты, комментарии, чаты и т.д.
- `id` - униальный id пользователя
- `uid` - уникальный идентификатор для внешнего использования
- `email` - почта
- `phone` - номер телефона
- `password_hash` - хэш пароля
- `is_active` - активный ли аккаунт
- `created_at` - дата создания
- `updated_at` - дата обновления
# user_profile
Таблица `userProfile` содержит дополнительную информацию о пользователе, которая не является критической для аутентификации, но важна для отображения профиля и взаимодействия с другими пользователями. Она связана с таблицей `profile` через `user_account_id`, что позволяет одному пользователю иметь несколько профилей (например, личный и бизнес).
- `id` - униальный id профиля пользователя
- `uid` - уникальный идентификатор для внешнего использования
- `user_account_id` - пользователь, которому принадлежит профиль
- `profile_id` - профиль из таблицы `profile`
- `first_name` - имя
- `last_name` - фамилия
- `bio` - описание профиля
- `birthday_date` - дата рождения
- `gender` - пол (`int` enum)
- `is_active` - активный ли профиль
- `created_at` - дата создания
- `updated_at` - дата обновления

# profile
Таблица `profile` представляет собой абстрактный профиль, который может быть связан с пользователем (через `userProfile`) или быть группой (через `community`). Это позволяет использовать единый интерфейс для взаимодействия с пользователем(человеком) или группой. Профиль содержит общую информацию, такую как имя пользователя и аватарка.
- `id` - уникальный id абстрактного профиля, единой точки для действий и контента
- `uid` - уникальный идентификатор для внешнего использования
- `avatar_id` - общая аватарка, отображаемая везде
- `username` - уникальное имя пользователя для упоминаний и ссылок
- `is_active` - активный ли аккаунт
- `created_at` - дата создания
- `updated_at` - дата обновления

# media
Таблица `media` содержит информацию о медиафайлах, которые могут быть прикреплены к постам, комментариям, сообщениям и другим сущностям. Она позволяет хранить метаданные о файлах, такие как размер, тип и описание, а также обеспечивает уникальную идентификацию каждого файла через `id`.
- `id` - униальный id файла
- `uid` - уникальный идентификатор для внешнего использования
- `media_name` - называния файла
- `extension` - расширение файла
- `mime_type` - MIME тип
- `description` - описание файла
- `size` - размер в байтах
- `is_active` - активный ли файл
- `created_at` - дата создания
- `updated_at` - дата обновления

# post
Таблица `post` содержит информацию о публикациях, которые создают пользователи или группы. Каждый пост связан с автором через `author_id`, который ссылается на абстрактный профиль. Это позволяет использовать единый механизм для отображения имени и аватарки автора, независимо от того, является ли он человеком или группой. Таблица также содержит поля для текста поста, дат создания и обновления, а также флаг активности, который может использоваться для мягкого удаления или скрытия постов.
- `id` - уникальный id поста
- `uid` - уникальный идентификатор для внешнего использования
- `post_text` - текст поста
- `author_id` - абстрактный профиль, создавший пост (ссылка на `profile`)
- `is_active` - активный ли пост
- `created_at` - дата создания
- `updated_at` - дата обновления

# post_with_media
Таблица `postWithMedia` служит для связи постов с медиафайлами, которые к ним прикреплены. Она позволяет одному посту иметь несколько медиафайлов и сохранять порядок их отображения.
- `post_id` - пост, к которому прикреплён медиафайл
- `media_id` - медиафайл поста
- `sort_order` - порядковый номер медиафайла в посте

# chat

Каждый чат имеет уникальный идентификатор, тип (личный или групповой), название и аватарку. Таблица также содержит даты создания и обновления, а также флаг удаления, который может использоваться для мягкого удаления чатов.
- `id` - униальный id чата
- `uid` - уникальный идентификатор для внешнего использования
- `chat_type` - тип чата
- `title` - название чата
- `avatar_id` - аватарка чата (media table)
- `is_active` - активный ли чат
- `created_at` - дата создания
- `updated_at` - дата обновления
# chat_member
Таблица `chatMember` содержит информацию об участниках чата, связывая их с абстрактными профилями и чатами. Она позволяет отслеживать, кто является участником какого чата, а также хранить информацию о дате вступления и выхода из чата, а также роли участника (например, админ, участник и т.д.).
- `id` - уникальный id участника чата
- `uid` - уникальный идентификатор для внешнего использования
- `chat_id` - чат
- `profile_id` - ссылка на `profile` (профиль или группа)
- `join_at` - дата вступления в чат
- `leave_at` - дата выхода из чата
- `is_active` - активный ли пользователь чата
- `created_at` - дата создания
- `updated_at` - дата обновления
- `chat_role` - роль в чате

# friendship
Таблица `friendship` содержит информацию о дружеских отношениях между пользователями. Таблица также содержит даты создания и обновления, что позволяет отслеживать историю дружеских отношений.
- `friend1_id` - первый участник (ссылка на `profile`)

- `requester_id` - id участника-инициатора дружбы (ссылка на `profile`)

- `is_active` - активна ли дружба
- `created_at` - дата создания
- `updated_at` - дата обновления

# session (REDIS)
Таблица `session` содержит информацию о сессиях пользователей, которые используются для аутентификации и управления доступом. Каждая сессия связана с пользователем через `profile_id`, что позволяет отслеживать, какие устройства и IP-адреса используются для доступа к аккаунту. Таблица также содержит информацию о дате создания, истечения срока действия и последней активности сессии.
- `id` - униальный id сессии
- `profile_id` - id пользователя, которому принадлежит сессия
- `ip` - IP-адрес, с которого была создана сессия
- `userAgent` - User-Agent, с которого была создана сессия
- `createdAt` - дата создания
- `expiresAt` - дата истечения срока действия
- `lastActivityAt` - дата последнего использования стикера
