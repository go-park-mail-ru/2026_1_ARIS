# user_account
Таблица `user_account` содержит базовую информацию о пользователе, необходимую для аутентификации и управления аккаунтом. Она служит основой для создания профиля пользователя и связывания его с различными аспектами социальной сети, такими как посты, комментарии, чаты и т.д.
- `id` - униальный id пользователя
- `email` - почта
- `phone` - номер телефона
- `password_hash` - хэш пароля
- `createdAt` - дата создания
- `updatedAt` - дата обновления
# userProfile
Таблица `userProfile` содержит дополнительную информацию о пользователе, которая не является критической для аутентификации, но важна для отображения профиля и взаимодействия с другими пользователями. Она связана с таблицей `profile` через `user_account_id`, что позволяет одному пользователю иметь несколько профилей (например, личный и бизнес).
- `id` - униальный id профиля пользователя
- `user_account_id` - пользователь, которому принадлежит профиль
- `profile_id` - профиль из таблицы `profile`
- `firstName` - имя
- `lastName` - фамилия
- `bio` - описание профиля
- `birthdayDate` - дата рождения
- `gender` - пол (`int` enum)
- `createdAt` - дата создания
- `updatedAt` - дата обновления

# profile
Таблица `profile` представляет собой абстрактный профиль, который может быть связан с пользователем (через `userProfile`) или быть группой (через `community`). Это позволяет использовать единый интерфейс для взаимодействия с пользователем(человеком) или группой. Профиль содержит общую информацию, такую как имя пользователя и аватарка.
- `id` - уникальный id абстрактного профиля, единой точки для действий и контента
- `avatar_id` - общая аватарка, отображаемая везде
- `username` - уникальное имя пользователя для упоминаний и ссылок
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `isActive` - активный ли аккаунт

# media
Таблица `media` содержит информацию о медиафайлах, которые могут быть прикреплены к постам, комментариям, сообщениям и другим сущностям. Она позволяет хранить метаданные о файлах, такие как размер, тип и описание, а также обеспечивает уникальную идентификацию каждого файла через `id`. Ссылка на файл (`link`) также является уникальной, что позволяет быстро находить медиа по URL.
- `id` - униальный id файла
- `name` - называния файла
- `extension` - расширение файла
- `description` - описание файла
- `link` - ссылка на файл
- `mimeType` - MIME тип
- `size` - размер в байтах
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `isDeleted` - удалено ли

# post
Таблица `post` содержит информацию о публикациях, которые создают пользователи или группы. Каждый пост связан с автором через `author_id`, который ссылается на абстрактный профиль. Это позволяет использовать единый механизм для отображения имени и аватарки автора, независимо от того, является ли он человеком или группой. Таблица также содержит поля для текста поста, дат создания и обновления, а также флаг активности, который может использоваться для мягкого удаления или скрытия постов.
- `id` - уникальный id поста
- `text` - текст поста
- `author_id` - абстрактный профиль, создавший пост (ссылка на `profile`)
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `isActive` - активный ли пост

# postWithMedia
Таблица `postWithMedia` служит для связи постов с медиафайлами, которые к ним прикреплены. Она позволяет одному посту иметь несколько медиафайлов и сохранять порядок их отображения.
- `post_id` - пост, к которому прикреплён медиафайл
- `media_id` - медиафайл поста
- `order` - порядковый номер медиафайла в посте

# chat
Таблица `chat` содержит информацию о чатах, которые могут быть как личными (между двумя пользователями), так и групповыми (с несколькими участниками). Каждый чат имеет уникальный идентификатор, тип (личный или групповой), название и аватарку. Таблица также содержит даты создания и обновления, а также флаг удаления, который может использоваться для мягкого удаления чатов.
- `id` - униальный id чата
- `type` - тип чата
- `title` - название чата
- `avatar_id` - аватарка чата (media table)
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `isDeleted` - удален ли чат
# chatMember
Таблица `chatMember` содержит информацию об участниках чата, связывая их с абстрактными профилями и чатами. Она позволяет отслеживать, кто является участником какого чата, а также хранить информацию о дате вступления и выхода из чата, а также роли участника (например, админ, участник и т.д.).
- `id` - уникальный id участника чата
- `chat_id` - чат
- `profile_id` - ссылка на `profile` (профиль или группа)
- `joinedAt` - дата вступления в чат
- `leaveAt` - дата выхода из чата
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `role` - роль в чате

# message
Таблица `message` содержит информацию о сообщениях, отправленных в чатах. Каждое сообщение связано с автором через `author_id`, который ссылается на абстрактный профиль, что позволяет использовать единый механизм для отображения имени и аватарки автора. Таблица также содержит текст сообщения, ссылку на родительское сообщение (для ответов), статус сообщения, а также флаг удаления.
- `id` - уникальный id сообщения
- `text` - текст сообщения
- `parentMessage_id` - id родительского сообщения (для ответов)
- `chat_id` - чат
- `status` - статус сообщения
- `sticker_id` - сообщение = стикер
- `author_id` - отправитель сообщения (ссылка на `profile`)
- `createdAt` - дата создания
- `updatedAt` - дата изменения
- `isDeleted` - удалено ли сообщение
# messageWithMedia
Таблица `messageWithMedia` служит для связи сообщений с медиафайлами, которые к ним прикреплены. Она позволяет одному сообщению иметь несколько медиафайлов и сохранять порядок их отображения.
- `message_id` - сообщение, к которому прикреплён медиафайл
- `media_id` - медиафайл сообщения
- `order` - порядковый номер медиафайла в сообщении

# community
Таблица `community` содержит информацию о группах, которые могут быть созданы пользователями. Каждая группа имеет уникальный идентификатор, название, описание, тип (публичная или приватная), а также ссылки на владельца группы и профиль группы. Таблица также содержит дату последнего обновления, которая может использоваться для сортировки групп по активности.
- `id` - униальный id группы
- `title` - название группы
- `bio` - описание группы
- `type` - тип группы (публичная, приватная)
- `owner_id` - профиль владельца группы (Profile table)
- `profile_id` - профиль группы (Profile table)
- `createdAt` - дата создания
- `updatedAt` - дата обновления

- `profile_id` - профиль участника группы
- `community_id` - группа
- `joinedAt` - дата вступления в группу
- `leaveAt` - дата покидания группы
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `role` - роль в группе (админ, участник и т.д.)

# friendship
Таблица `friendship` содержит информацию о дружеских отношениях между пользователями. Она связывает два абстрактных профиля (которые могут быть как людьми, так и группами) и хранит статус их дружбы (например, запрос, принята, отклонена). Таблица также содержит даты создания и обновления, что позволяет отслеживать историю дружеских отношений.
- `friend1_id` - первый участник (ссылка на `profile`)
- `friend2_id` - второй участник (ссылка на `profile`)
- `status` - статус дружбы (запрос, принята, отклонена)
- `createdAt` - дата создания
- `updatedAt` - дата обновления

# stickerpack
Таблица `stickerpack` содержит информацию о стикерпаке, который может быть создан пользователями. Каждый стикерпак имеет уникальный идентификатор, название, а также ссылку на автора (который может быть как человеком, так и группой). Таблица также содержит даты создания и обновления, а также флаг удаления, который может использоваться для мягкого удаления стикерпака.
- `id` - униальный id стикерпака
- `title` - название стикерпака
- `author_id` - кто создал стикерпак (ссылка на `profile`) (опционально)
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `isDeleted` - удален ли стикерпак
# sticker
Таблица `sticker` содержит информацию о стикере, который может быть частью стикерпака. Каждый стикер имеет уникальный идентификатор, ссылку на изображение, размер в байтах, а также порядковый номер в стикерпаке. Таблица также содержит ссылку на стикерпак, которому принадлежит стикер, дату создания и флаг удаления.
- `id` - униальный id стикера
- `link` - ссылка на стикер
- `size` - размер стикера в байтах
- `indexOrder` - порядковый номер стикера в паке
- `pack_id` - пак, которому пренадлежит стикер
- `createdAt` - дата создания
- `updatedAt` - дата обновления
- `isDeleted` - удален ли стикер

# session
Таблица `session` содержит информацию о сессиях пользователей, которые используются для аутентификации и управления доступом. Каждая сессия связана с пользователем через `profile_id`, что позволяет отслеживать, какие устройства и IP-адреса используются для доступа к аккаунту. Таблица также содержит информацию о дате создания, истечения срока действия и последней активности сессии.
- `id` - униальный id сессии
- `profile_id` - id пользователя, которому принадлежит сессия
- `ip` - IP-адрес, с которого была создана сессия
- `userAgent` - User-Agent, с которого была создана сессия
- `createdAt` - дата создания
- `expiresAt` - дата истечения срока действия
- `lastActivityAt` - дата последнего использования стикера

# adMeta
Таблица `adMeta` содержит метаданные, связанные с рекламными постами. Она позволяет хранить дополнительные ключ-значение пары, которые могут быть использованы для таргетинга, аналитики или других целей, связанных с рекламой. Каждая запись в этой таблице связана с конкретной рекламой через `ad_id`.
- `id` - уникальный id метаданных
- `ad_id` - реклама, к которой относятся метаданные
- `key` - ключ метаданных
- `value` - значение метаданных
- `createdAt` - дата создания
- `updatedAt` - дата обновления
